Class Hierarchy

  • DrawableObject.DrawableObject: This is the super class of all object that can be drawn to the screen
  • Entity.Locals
  • Mouse.Mouse: A class which keeps track of mouse events and processes them to determine when a mouse is being clicked or dragged, and the associated buttons and positions.
  • Event.MouseEvent.MouseLocals
  • object: The most base type
    • ControlMapper.ControlMapper: Reads from a configuration file in order to establish a control scheme.
    • Debugger.Debugger: The debugger is used for debugging purposes during development.
    • Event.Event'.Event: This is a superclass for any events that can be read by the low level event manager
      • Event.GenericDebugEvent'.GenericDebugEvent: Fire this event instead of using a print statement for debugging purposes.
      • Event.MouseEvent.MouseClickedEvent: Basic mouse event.
      • Event.MouseEvent.MouseMovedEvent: Fired when the mouse moves; keeps track of position only.
      • Event.QuitEvent'.QuitEvent: Request to close the program
      • Event.RefreshCompleteEvent'.RefreshCompleteEvent: Called when drawing to the screen is completed
      • Event.RefreshEvent'.RefreshEvent: Request to update the screen
      • Event.RenderEvent'.RenderEvent: Request to draw to the display surface
      • Event.StartEvent'.StartEvent: Request to begin monitoring input in the window
      • Event.UpdateEvent'.UpdateEvent: Request to update objects.
      • Event.WorldChangeEvent'.WorldChangeEvent: Request to change world.
    • Event.EventTimer'.EventTimer: This object is in charge of always knowing what time it is with respect to server-client synchronization
    • Grid.Grid
      • Grid.FiniteGrid: A standard, finite grid (used in most RTS games).
      • Grid.InfiniteGrid: A grid that functions like a torus - go off one end and come back on the other side
    • Listener.Listener: A Listener waits for events sent out by the manager to which it is registered.
      • Renderer.Renderer: A renderer simply draws object to the screen.
      • Universe.Universe
      • UserInterface.UserInterface: Contains all of the functionality required to present information screens to the user and interprets user input.
      • Window.Window: A wrapper for a pygame screen, providing easily accessible functionality such as setting the window resolution and enabling full screen mode.
    • Manager.Manager: All communication between the Universe, UserInterface, and Window happens here.
    • Player.Player: A Player object contains all of the information needed for entity ownership and functions associated with an individual and all of their units.
    • Screen.Screen: Contains all of the elements that might be shown on screen at once.
      • Screen.MainScreen: The screen where all of the action takes place.
    • pygame.sprite.Sprite: simple base class for visible game objects pygame.sprite.Sprite(*groups): return Sprite
    • Grid.TMP_Terrain
    • Viewport.Viewport: Acts as a "window" into the world it contains.
    • World.World: A World is an object that contains everything in the current environment that the player is able to see on the map by scrolling around.
